# Kairos Parts — Day 8 Client Vehicles, Documents and Repeat Requests

## Що реалізовано

Day 8 розширив кабінет клієнта як постійну робочу зону:

- розділ `Мій парк техніки`;
- створення одиниці техніки;
- редагування одиниці техніки;
- створення заявки по конкретній техніці;
- повторна заявка на основі попередньої заявки;
- розділ `Документи`;
- API для client vehicles і client documents;
- prefill логіка для `/request?source=client&vehicleId=<id>`;
- prefill логіка для `/request?source=client&repeatRequestId=<id>`.

## Мій парк техніки

Сторінка `/client/vehicles` показує тільки техніку поточного `ClientProfile`.

Для кожної одиниці техніки показується:

- тип техніки;
- виробник;
- модель;
- рік;
- VIN / серійний номер;
- коментар;
- кількість пов'язаних заявок;
- посилання `Детальніше`;
- CTA `Створити заявку по цій техніці`.

Якщо техніки немає, показується empty state з кнопкою `Додати техніку`.

## Поля техніки

Обов'язкові:

- тип техніки;
- виробник;
- модель.

Необов'язкові:

- рік;
- VIN / серійний номер;
- коментар.

Фото або документи до техніки не завантажуються напряму на Day 8. Документи відображаються через існуючі `RequestFile` і `Document` записи.

## Створення і редагування техніки

Створення:

- `/client/vehicles/new`;
- server action `createVehicle`;
- прив'язка до поточного `ClientProfile`;
- після створення редірект на `/client/vehicles?created=1`.

Редагування:

- `/client/vehicles/[id]`;
- server action `updateVehicle`;
- пошук тільки через `{ id, clientId }`;
- якщо техніка не належить клієнту, повертається 404.

## Заявка по конкретній техніці

Кнопки ведуть на:

```txt
/request?source=client&vehicleId=<id>
```

Якщо `vehicleId` належить поточному CLIENT:

- підставляється тип техніки;
- підставляється виробник;
- підставляється модель;
- підставляється VIN / серійний номер;
- підставляються контактні дані клієнта;
- після submit заявка створюється з `vehicleId`;
- `source = CLIENT_DASHBOARD`.

Якщо `vehicleId` не належить клієнту, prefill не виконується і заявка не прив'язується до чужої техніки.

## Повторна заявка

На `/client/requests` і `/client/requests/[id]` додано кнопку `Повторити`.

Кнопка веде на:

```txt
/request?source=client&repeatRequestId=<id>
```

Якщо заявка належить поточному CLIENT:

- підставляється категорія;
- тип техніки;
- виробник;
- модель;
- VIN / серійний номер;
- опис потреби;
- пов'язана техніка, якщо вона була у попередній заявці;
- контактні дані клієнта.

Файли з попередньої заявки не копіюються автоматично. Нова заявка отримує новий `requestNumber` і новий `publicStatusToken`.

## Документи

Сторінка `/client/documents` показує:

- `RequestFile`, прикріплені до заявок клієнта;
- `Document`, які прив'язані до клієнта, заявки клієнта або техніки клієнта.

Для кожного файлу показується:

- назва;
- mimeType;
- розмір;
- дата завантаження;
- пов'язана заявка;
- пов'язана техніка;
- статус доступу.

Прямі приватні storage paths не відкриваються як публічні download links.

## API

Реалізовано:

- `GET /api/client/vehicles`;
- `POST /api/client/vehicles`;
- `GET /api/client/vehicles/[id]`;
- `PATCH /api/client/vehicles/[id]`;
- `GET /api/client/documents`.

API перевіряють:

- наявність session;
- `role = CLIENT`;
- поточний `ClientProfile`;
- ownership через `clientId`.

## Зв'язки

Після Day 8 працюють зв'язки:

- клієнт → техніка через `Vehicle.clientId`;
- клієнт → заявки через `Request.clientId`;
- техніка → заявки через `Request.vehicleId`;
- заявка → файли через `RequestFile.requestId`;
- заявка → документи через `Document.requestId`;
- техніка → документи через `Document.vehicleId`.

## Що працює після Day 8

- Клієнт може вести парк техніки.
- Клієнт може редагувати свою техніку.
- Клієнт може створити заявку по конкретній техніці.
- Клієнт може повторити власну заявку.
- Клієнт бачить свої файли і документи.
- CRM у майбутньому зможе бачити, по якій техніці створена заявка.

## Навмисно не реалізовувалось

- CRM manager UI.
- Telegram flow.
- OCR execution.
- Комерційні пропозиції в системі.
- Погодження КП.
- Історія комунікації.
- Розширені Telegram-сценарії.
- Оплата, магазин, кошик.
- Viber, Нова пошта, BAS/ERP.
- Кабінет постачальника.
- Автоматичне копіювання файлів при повторній заявці.
- Публічне відкриття приватних storage paths.
